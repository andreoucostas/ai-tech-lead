ONE-PAGE TEAM OPERATING REFERENCE

What happens when the team asks the AI to code?

Running example: “Add CSV export to the dashboard. Only admins can export.” Follow the real events, files, machine actions, evidence, and human decisions.

01 · INSTALL

Framework enters the repo

install.ps1/.sh detects .NET, Angular, or mixed; selects dist/*; archives collisions; copies files. Run /bootstrap to replace generic context.

HUMAN: review archive + commit
02 · START

Rules and live state load

CLAUDE.md/AGENTS.md supplies stable rules. SessionStart → session-start.* injects branch, commits, debt, security, wiki, and hazards.

CONTEXT: client support varies
03 · ROUTE

Prompt selects methodology

UserPromptSubmit → route-prompt.* classifies Feature plus Security overlay and injects the plan gate. Explicit /feature skips classification.

HUMAN: clarify acceptance
04 · PLAN

Team inspects intended work

Assistant names files, order, reused patterns, failure modes, tests, debt, and assumptions. Non-trivial unresolved design moves to /design.

HUMAN: approve/correct scope
05 · WRITE

Each editor change is guarded

PreToolUse → guard.* allows clean content or blocks defined secrets, suppressions, focused/skipped tests, and tautologies. Shell writes are outside this floor.

HARD BLOCK: supported clients
06 · FEEDBACK

Write triggers build and audit

PostToolUse → post-write.* + audit-trail.*. Fast compile/type feedback reaches supported clients; timestamp, branch, and relative file path append to the audit log.

MACHINE ACTION + TRACE
07 · VERIFY

Coherent subtask is proven

Run targeted tests after service/query, auth tests after endpoint/policy, then build and relevant regression scope. Name commands and observed results.

DETERMINISTIC RESULT
08 · REVIEW

Agent and focused lenses inspect

Self-review checks conventions and drift. SOLID/convention/debt agents advise. Boy Scout hook reports bounded cleanup candidates with deduplication.

ADVISORY: can miss defects
09 · HUMAN

Reviewer tests understanding

Check business rules, authorization, CSV injection/scale, architecture, test sensitivity, and whether the developer can explain choices and evidence.

HUMAN AUTHORITY
10 · CI

Independent gates rerun

Build, tests, analyzers, architecture, documentation, and configured framework scripts run outside the assistant session. Wiring must be blocking to be a gate.

PLATFORM-OWNED ACCEPTANCE
11 · RECORD

Project truth stays current

Update architecture/ADRs for decisions, conventions for stable rules, learnings for recurring corrections, and debt/security registers for unfinished risks.

TEAM MEMORY
12 · MERGE

An accountable owner decides

Local evidence + CI + review inform merge/release. Models and hooks do not own correctness, residual risk, exceptions, or production outcomes.

Who owns what

DeveloperAcceptance criteria, plan review, diff understanding, evidence inspection, honest handoff.
ReviewerCorrectness, risk, architecture, test meaning, and developer understanding.
Tech leadCanonical rules, architecture/defaults, exceptions, ADR/debt/security quality.
PlatformInstall/update, client prerequisites, diagnostics, CI wiring, blocking merge gates.
Product/securityBusiness policy, threat/compliance constraints, acceptance of relevant residual risk.
First machine check: run scripts/framework-doctor.ps1 or .sh. Do not assume hook support merely because the files exist.
Framework does: load rules/context, route process, block defined editor patterns, run configured checks, preserve evidence, and expose drift.
Framework does not: invent missing requirements, semantically prove all code, make every client equivalent, or replace CI and review.
Failure rule: stop on material ambiguity or failed evidence. Fix the cause; disclose what was not verified; give every exception an owner.
